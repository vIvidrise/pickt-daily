# 실서비스 → 지금 버전 업데이트 요약

**기준:** pickt-daily.vercel.app (실서비스) → 현재 로컬 코드베이스

---

## 1. 오늘 내 운세 (Fortune)

### 오늘 내 운세 배너 (홈)
- **유지:** 배너 이미지 (`fortune-banner.png`) 사용
- **변경:** 배너 이미지 파일 교체 (비비 캐릭터 + 크리스탈볼, 보라·오렌지 배경)
- **파일:** `public/fortune-banner.png`

### 운세 결과 화면 (FortuneResult) — 신규 기능
| 항목 | 내용 |
|------|------|
| **위치 권한 팝업** | 결과 진입 시 비비 캐릭터와 함께 위치 허용 요청 모달 표시 |
| **행운 장소 섹션** | 권한 허용 시 `lucky_food` 기반 근처 맛집/장소 추천 (반경 1km) |
| **카드 UI** | Swiper 가로 스와이프, Glassmorphism 스타일, 상호명·거리·혼밥 지수 노출 |
| **비비 캐릭터** | `solo_difficulty_level`(1~5)에 따라 카드 우측 하단에 캐릭터 배치 |

**신규/수정 파일:**
- `LocationPermissionModal.jsx` / `.css` — 위치 권한 커스텀 팝업
- `LuckyPlaceSwiper.jsx` / `.css` — 행운 장소 가로 스와이프 카드
- `FortuneResult.jsx` / `.css` — 모달·권한·행운 장소 섹션 통합

## 2. API

### `fetchLuckyPlacesNearby` (gemini.js)
- **역할:** `lucky_food`(예: 순대국) + 유저 좌표로 반경 1km 이내 장소 검색
- **필터:** `solo_difficulty_level` 있는 장소 우선
- **반환:** 상호명, 거리(m/km), 혼밥 지수, 네이버 플레이스 링크

---

## 3. 홈 (Home)

## 5. 배포 후 확인 방법

1. **홈** — 오늘 내 운세 배너가 새 이미지로 보이는지
2. **운세** → 이름/생년월일 입력 → 운세보기
3. **운세 결과** — 위치 권한 모달 노출 여부
4. **위치 허용** — 행운 장소 카드 섹션 표시 여부
5. **위치 나중에** — 행운 장소 섹션 미표시 여부
