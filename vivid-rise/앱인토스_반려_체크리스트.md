# 앱인토스 반려 사항 체크리스트 (실서비스·제출용)

제출 전 아래 4가지 반려 사유가 모두 해결되었는지 확인하세요.

---

## 1. 자체 구현한 헤더 제거 — ✅ 코드 반영

- **반려:** 자체 구현한 헤더는 제거 부탁드려요.
- **조치:** 앱인토스에서 실행될 때(`use-toss-nav`) 모든 커스텀 헤더를 **CSS로 비표시** 처리했습니다.
  - 숨김 대상: `home-header`, `confirm-header`, `saved-header`, `fortune-header`, `fortune-result-header`, `map-header`
- **파일:** `src/App.css`

---

## 2. 공통 내비게이션 바: 브랜드 로고 + 미니앱 이름 '요즘 뭐 함' — ✅ 설정 반영, 콘솔 확인 필요

- **반려:** 공통 내비게이션 바 브랜드 로고가 누락되었어요. 미니앱 이름은 앱 정보와 동일한 **'요즘 뭐 함'**으로 수정해 주세요.
- **코드 조치:**
  - `granite.config.ts` / `app.config.js`에 `brand.displayName: '요즘 뭐 함'`, `brand.icon`(로고 URL), `navigationBar` 설정 반영.
- **콘솔에서 확인:**
  1. [앱인토스 개발자센터](https://developers-apps-in-toss.toss.im/) → 해당 앱 선택.
  2. **앱 정보**에서 **앱 이름(표시 명칭)** 을 **요즘 뭐 함**으로 저장.
  3. **앱 아이콘(브랜드 로고)** 이미지를 업로드한 뒤, 해당 이미지 **우클릭 → 링크 복사** 한 URL을 `granite.config.ts`의 `brand.icon`에 넣어 두었는지 확인. (로고가 안 보이면 이 URL을 콘솔/설정과 맞춰 주세요.)

---

## 3. 운세 추천 식당 터치 시 선택한 식당 노출 — ✅ 코드 반영

- **반려:** 운세에 추천 식당을 터치하고 네이버 지도로 연결되지만, 선택한 식당이 노출되지 않아요.
- **조치:** 앱인토스 WebView에서는 네이버 지도 URL을 **`openURL`(openExternalUrl)** 로 열어, 외부 브라우저/네이버 앱에서 해당 장소 상세가 바로 보이도록 수정했습니다.
- **파일:** `src/utils/naverMapScheme.js`

---

## 4. 앱 내 기능 스킴 최소 1개 등록 — ✅ 안내 반영, 콘솔에서 등록 필요

- **반려:** 앱 내 기능 스킴을 최소 한 개 이상 등록해 주세요.
- **조치:** 개발자센터 콘솔에서 **기능 스킴** 또는 **앱 진입 스킴/딥링크** 메뉴를 찾아 아래 중 **최소 1개 이상** 등록해 주세요.
  - **진입 URL:** 미니앱 웹 주소 (예: `https://vivid-rise.vercel.app/` 또는 NCP 웹 서비스 URL)
  - **앱 스킴:** 예: `intoss://pickt-daily` (콘솔 가이드에 따라 appName과 일치하게 등록)
- **참고:** `앱인토스_반려대응_UI_헤더.md` §4.
