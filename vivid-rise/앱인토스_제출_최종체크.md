**제출용 진입 URL:** `https://pickt-daily.vercel.app` (또는 `https://pickt-daily.vercel.app/`)

---

## ✅ URL·배포 상태 (괜찮음)

| 항목 | 상태 |
|------|------|
| 실서비스 주소 | **https://pickt-daily.vercel.app** — vivid-rise 프로젝트에 연결됨 |
| 배포된 버전 | 최신 (9개 지역, 지도·길찾기, 운세 UI, 주소 표시 등 반영) |
| 앱인토스 진입 URL로 사용 | ✅ 이 주소 그대로 제출 가능 |

---

## ✅ 앱인토스 비게임 체크 (이미 반영됨)

- 라이트 모드, 핀치줌 비활성화, 접근성(aria-label, 44px 터치)
- Safe Area·내비게이션 연동 (뒤로가기, 홈, 찜 버튼)
- **지도·길찾기·네이버 지도 연동**: 실제 좌표·주소, 길찾기 버튼, 네이버 플레이스 링크, fitBounds — 실기기에서 동작 확인됨
- 지도 실패 시 Leaflet(OpenStreetMap) 자동 전환

## ⚠️ 제출 전 꼭 확인할 것

### 1. 네이버 지도가 실서비스·테스트에서 동작하려면

- **네이버 클라우드플랫폼(NCP)** → [콘솔](https://console.ncloud.com) → **Services** → **Application Services** → **Maps** → 사용 중인 **Application** 선택
- **Application 수정**에서 **웹 서비스 URL**에 아래 주소를 **모두 등록**하세요:

  | 용도 | 등록할 URL |
  |------|------------|
  | 실서비스(웹) | `https://pickt-daily.vercel.app` |
  | 실서비스(웹, 슬래시) | `https://pickt-daily.vercel.app/` |
  | **앱인토스 테스트(QR/ intoss-private)** | `https://pickt-daily.private-apps.tossmini.com` |
  | 앱인토스 출시 후 실제 | `https://pickt-daily.apps.tossmini.com` |

- **테스트(intoss-private)에서 "인증이 실패했습니다"**가 나오면 → **`https://pickt-daily.private-apps.tossmini.com`** 이 NCP 웹 서비스 URL에 없어서입니다. 위 표의 테스트용 URL을 추가한 뒤 저장하면 됩니다.
- 등록 후 변경 사항 반영까지 잠시 걸릴 수 있으니, 1~2분 뒤 앱 다시 열어서 확인하세요.

### 2. 앱인토스 콘솔 설정

- [앱인토스 개발자센터](https://developers-apps-in-toss.toss.im/) → 해당 앱 선택
- **진입 URL**(또는 미니앱 URL)을 **https://pickt-daily.vercel.app/** 로 설정
- .ait 번들 업로드는 별도 (필요 시 `npm run build` 후 생성된 .ait 업로드)

### 3. 실제 기기에서 한 번씩 확인 권장

- **https://pickt-daily.vercel.app** 접속 후
  - 오늘 뭐 먹지 / 오늘 뭐 하지: 지역 9개 선택 → 결과 → 지도·길찾기·주소
  - 오늘 내 운세: 입력 → 결과 → 행운 장소 리스트
- 토스 앱 WebView에서 열었을 때 내비게이션·Safe Area가 자연스러운지 확인
