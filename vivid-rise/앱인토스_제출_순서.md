# 앱인토스 제출 순서

아래 **순서대로** 진행하면 됩니다.

---

## 1단계: 제출 전 확인 (한 번만)

| 순서 | 할 일 | 확인 |
|------|--------|------|
| 1-1 | **NCP(네이버 지도)** 콘솔 → Maps → 웹 서비스 URL에 `https://pickt-daily.vercel.app` / `https://pickt-daily.vercel.app/` 등록 | ☐ |
| 1-2 | **실기기**에서 https://pickt-daily.vercel.app 접속 → 오늘 뭐 먹지·오늘 뭐 하지·운세, 지도·길찾기 동작 확인 | ☐ |

---

## 2단계: .ait 빌드 (최신 버전으로)

```bash
cd /Users/ad/Desktop/pickt-daily/vivid-rise
npm install --legacy-peer-deps
npm run build
```

- 끝나면 **pickt-daily.ait** 파일이 `vivid-rise` 폴더에 생성됨.
- `pickt-daily.ait 빌드 완료` 메시지 확인.

---

## 3단계: 앱인토스 개발자센터 접속

1. **[앱인토스 개발자센터](https://developers-apps-in-toss.toss.im/)** 접속 후 로그인.
2. 제출할 **앱**(예: pickt-daily) 선택.

---

## 4단계: 진입 URL 설정

1. 콘솔에서 해당 앱의 **앱 정보** 또는 **설정** 메뉴로 이동.
2. **진입 URL**(또는 미니앱 URL)을 **https://pickt-daily.vercel.app/** 로 설정 후 저장.

---

## 5단계: 앱 번들(.ait) 업로드

1. 콘솔에서 **앱 번들 업로드**(또는 **출시** → **번들 업로드**) 메뉴로 이동.
2. 2단계에서 만든 **pickt-daily.ait** 파일 선택 후 업로드.
3. 업로드 완료까지 대기.

---

## 5-2단계: 토스앱에서 동작 확인 (테스트하기)

번들 업로드가 끝나면 **토스앱 안에서** 미니앱이 잘 도는지 확인하는 단계예요.

1. **앱인토스 콘솔** → 워크스페이스 선택 → 해당 앱 선택 → 왼쪽 메뉴 **앱 출시**.
2. 업로드한 번들 옆의 **"테스트하기"** 버튼 클릭.
3. 화면에 나오는 **테스트용 QR 코드**를 **토스앱**으로 스캔.
4. 토스앱이 열리면 **미니앱이 실행**됩니다. (오늘 뭐 먹지·오늘 뭐 하지·운세, 지도·길찾기 등 동작 확인.)
5. **검토 요청**을 하려면 이 테스트를 **최소 1회 이상** 완료해야 합니다.

**테스트 가능 조건**

- 토스앱에 **로그인**된 상태
- 해당 앱의 **워크스페이스 멤버**
- **만 19세 이상** 계정

**참고:** [토스앱에서 테스트](https://developers-apps-in-toss.toss.im/development/test/toss.html)

---

## 6단계: 검토 요청 → 출시

1. **검토 요청하기** 버튼 클릭 (있는 경우).
2. 앱인토스 검수 승인 후 **출시하기**로 배포 완료.

---

## 요약 체크리스트

| 단계 | 내용 |
|------|------|
| 1 | NCP 웹 서비스 URL 등록 + 실기기 동작 확인 |
| 2 | `npm run build` → pickt-daily.ait 생성 |
| 3 | 앱인토스 개발자센터 로그인 → 앱 선택 |
| 4 | 진입 URL = https://pickt-daily.vercel.app/ |
